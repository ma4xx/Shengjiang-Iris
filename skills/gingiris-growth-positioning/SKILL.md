# Gingiris Growth Positioning

## Goal

把产品讲成用户一眼懂、还能和竞品区分开的故事。

## Iris Judgment Layer

定位输出默认遵循这几个判断：

- 一句话先让全球目标用户听懂，再谈创意
- 对标竞品不是减分，而是帮助确认用户认知
- 先讲用户问题和比较性优势，不先讲功能清单
- 如果一句话还讲不清，就说明场景太宽，不是文案太弱

## Use This Skill When

- 需要一句话介绍
- 需要官网首屏文案
- 需要竞品对比
- 需要 hero section、tagline、subheadline
- 需要把技术表达改成用户表达

## Core Rules

### 1. 先讲价值，不先讲技术

用户先关心：

- 解决什么问题
- 适合谁
- 比竞品强在哪
- 为什么现在要用

技术细节只在能证明价值时出现。

### 2. 10 秒可懂

一句话必须满足：

- 目标用户清楚
- 痛点清楚
- 价值清楚
- 差异化清楚

### 3. 先对标，再抽象

优先找 3 个最像的竞品并排看，再写自己的表达。

同时回答这两个问题：

- 用户为什么先想到竞品，不先想到你
- 你借竞品的哪部分认知，又明确切掉了哪部分误解

## Output Format

给出这 4 层：

1. 一句话介绍
2. 竞品参照系
3. Hero 标题
4. 副标题
5. 三个支撑点

## Useful Templates

- `我们帮助 [谁] 解决 [什么问题]，通过 [独特方式]，在 [结果] 上更快或更稳。`
- `不是 [旧方案]，而是 [新方案]。`

## Guardrails

- 不要写空泛口号
- 不要堆功能名
- 不要把所有卖点都放进首屏
- 如果一句话讲不清，就先收窄场景
- 不要为了“显得独特”而故意不对标竞品

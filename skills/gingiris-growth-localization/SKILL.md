# Gingiris Growth Localization

## Goal

把产品改造成当地用户能理解、能使用、能付费的版本。

## Iris Judgment Layer

本地化不是简单翻译。先问：

- 这个市场用户是不是已经听懂你的价值主张
- 问题出在语言，还是出在 workflow / 支付 / 信任 / 渠道
- 现在应该先做小试点，还是已经具备放大条件

## Use This Skill When

- 要选首发市场
- 要做语言适配
- 要做支付和结算
- 要考虑合规和数据存储
- 要判断某个国家能不能直接复制中国做法

## Core Rules

### 1. 市场不是翻译

语言只是第一层，真正要改的是：

- workflow
- 支付
- 渠道
- 客服
- 合规

### 2. 先看当地差异

至少判断这几项：

- 用户购买习惯
- 主要渠道
- 本地竞争强度
- 法规和数据要求
- 是否需要本地伙伴

### 3. 先小范围试

先选一个国家或区域做验证，不要一口气铺开。

## Output Format

输出时给：

1. 目标市场
2. 当前市场判断
3. 适配重点
4. 风险点
5. 本地化优先级
6. 渠道与合作建议
7. 最小试点方案

## Guardrails

- 不要默认美国经验能直接搬走
- 不要只翻译文案不改流程
- 不要忽略支付和合规
- 不要在不知道差异的情况下扩大市场
- 不要把价值主张没讲懂的问题误判成本地化问题

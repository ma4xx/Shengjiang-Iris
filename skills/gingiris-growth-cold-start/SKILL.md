# Gingiris Growth Cold Start

## Goal

把一个产品从“没有信号”推到“有第一批真实用户和真实反馈”。

## Iris Judgment Layer

进入具体动作前，先做这 4 个判断：

- 这件事有没有一句全球用户能懂的话
- 现在最该拿的是前 10 个高质量用户，还是更多泛流量
- 这次动作能不能直接帮助跑通付费或传播闭环
- 产品、内容、渠道、用户路径是不是被一起设计

## Use This Skill When

- 需要找第一批用户
- 需要设计冷启动获客路径
- 需要写 DM / 邮件 / 私信文案
- 需要定义 beta 用户、激活指标、早期转化目标
- 需要判断现在该继续试还是该停

## Core Process

### 1. 定义一件事

先锁定一个 ICP 和一个主场景。

- 不要同时打多个受众
- 不要同时验证多个价值主张
- 不要先做大而全的品牌叙事
- 不要还没讲清楚价值主张就急着铺渠道

### 2. 找第一批信号

优先用最短路径：

- 竞品用户 / 竞品粉丝
- Reddit / X / LinkedIn 的高意图帖子
- 社媒关键词回复
- 直接 DM / 邮件 / 私信
- beta 群、Discord、WhatsApp、Telegram

优先顺序不是“哪里热闹去哪里”，而是“哪里最可能出现高意图用户，哪里最容易拿到真实回复”。

### 3. 跑小闭环

把路径拆成可数的动作：

- 触达多少人
- 有多少人回复
- 有多少人愿意看 demo
- 有多少人进入产品
- 有多少人激活
- 有多少人付费或承诺继续使用

如果用户没有付费入口，就先定义传播闭环，例如回复、预约 demo、愿意引荐朋友、愿意写 testimonial。

### 4. 只保留有效动作

保留带来真实反馈的动作，砍掉只带来噪音的动作。

## Output Format

输出时至少给这 5 项：

1. 目标 ICP
2. 一句话价值主张
3. 第一批用户来源
4. 具体触达话术
5. 7 天行动表
6. 验收指标和停止条件

## Guardrails

- 访谈不要做成 investor pitch
- 不要把“热闹”当成“验证”
- 不要在没有反馈前扩大预算
- 没有信号时，先换场景和话术，再换渠道
- 不要把 Product Hunt、Reddit、SEO 写成固定套餐，要按产品和阶段选择

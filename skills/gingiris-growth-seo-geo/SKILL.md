# Gingiris Growth SEO GEO

## Goal

把内容和页面做成既能被搜索到，也能被 AI 引用。

## Iris Judgment Layer

SEO / GEO 不是单纯追词，而是把“全球都能懂的价值主张”和“长期可复用内容资产”变成稳定入口。

- 先看页面服务哪个用户问题
- 先看页面有没有承接动作
- 先看首页、Readme、帮助中心、教程能不能互相回流

## Use This Skill When

- 要做 SEO
- 要做 GEO / AI 搜索可见性
- 要写博客、落地页或 FAQ
- 要处理 canonical、内链、孤岛页
- 要做 llms.txt、Schema、程序化 SEO

## Core Rules

### 1. 页面不是孤岛

所有重要页面都要有入口和回流路径。

### 2. 先给直接答案

页面开头先写一句能被抽取的直接答案，再展开。

### 3. 写可引用的内容

优先放：

- 明确定义
- 清晰步骤
- 数据表
- FAQ
- 场景化结论
- 明确的下一步动作

## Output Format

输出时给：

1. 目标关键词或主题簇
2. 页面服务的价值主张
3. 页面结构
4. 内链方案
5. FAQ / Schema 建议
6. canonical / llms.txt 建议
7. 验收检查表

## Guardrails

- 不要只写长文，不管结构
- 不要让页面互相抢词
- 不要忽略首页到内页的链接
- 不要只追流量，不看转化
- 不要把 SEO 当成和产品、文档、帮助中心分开的事

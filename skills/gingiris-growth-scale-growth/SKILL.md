# Gingiris Growth Scale Growth

## Goal

把已经跑通的增长动作，变成可重复、可放大的系统。

## Iris Judgment Layer

只有同时满足下面两条，才进入放大：

- 价值主张已经相对稳定
- 某个渠道或闭环已经证明有转化，不只是有曝光

否则，先退回 `positioning`、`cold-start` 或 `user-research`。

## Use This Skill When

- 已经有 PMF 或接近 PMF
- 要放大某个渠道
- 要做广告、KOL、红人、联盟、内容投放
- 要设计 campaign 和漏斗
- 要分配预算、看回报

## Core Rules

### 1. 先小后大

先用小样本测试，再决定是否放大。

### 2. 看漏斗，不看热闹

每个渠道都要能落到：

- 看见
- 点击
- 进站
- 注册
- 激活
- 付费

如果是 B2B 或高客单产品，要把“预约 demo / 回复邮件 / 进入试点”也视为关键转化动作。

### 3. 内容包先行

无论是 KOL、广告还是合作渠道，都先准备内容包：

- 一句话介绍
- 价值主张
- 关键卖点
- 示例截图或视频
- 跳转链接

## Output Format

输出时给：

1. 目标渠道
2. 当前已验证的闭环
3. 测试假设
4. 内容包
5. 预算建议
6. 漏斗指标
7. 放大或停止条件

## Guardrails

- 不要一上来铺太多渠道
- 不要只看曝光
- 不要不看转化就继续加预算
- 不要把早期尝试当成长期结构
- 不要把“规模化增长”当成修基本盘的方法

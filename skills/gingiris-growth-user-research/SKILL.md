# Gingiris Growth User Research

## Goal

把用户说的话，整理成能驱动产品决策的信号。

## Iris Judgment Layer

访谈的目标不是“多聊一点”，而是搞清楚：

- 用户是不是听懂了你的价值主张
- 用户真实工作流里最痛的一步是什么
- 用户为什么选竞品，为什么不选你
- 用户在什么情况下愿意付费、传播或继续回来

## Use This Skill When

- 要做早期访谈
- 要找 beta 用户
- 要写访谈提纲
- 要观察用户实际怎么用产品
- 要把反馈分成可行动的问题

## Interview Rules

### 1. 不是 pitch

访谈不是向用户推销。

要问的是：

- 你怎么开始用的
- 你现在怎么做
- 你为什么会换工具
- 哪一步最烦
- 什么情况下你愿意付费

### 2. 先看真实行为

优先让用户：

- 共享屏幕
- 演示当前工作流
- 说出正在用的竞品
- 说明卡点和替代方案

### 3. 只抓决策信号

把反馈分成这几类：

- 明确痛点
- 竞品替代
- 使用场景
- 付费信号
- 激活信号
- 留存风险
- 价值主张是否听懂

## Output Format

输出时给：

1. 访谈目标
2. 当前要验证的判断
3. 用户筛选条件
4. 问题清单
5. 记录模板
6. 反馈结论
7. 下一步行动

## Guardrails

- 不要问成问卷
- 不要只记“喜欢/不喜欢”
- 不要忽略竞品
- 不要把单次反馈当成整体结论
- 不要只收集态度，不追真实行为

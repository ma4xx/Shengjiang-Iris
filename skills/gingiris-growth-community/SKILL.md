# Gingiris Growth Community

## Goal

把用户、伙伴、内容和渠道组织成能长期复用的增长系统。

## Iris Judgment Layer

社区默认不是为了“显得热闹”，而是为了：

- 更快拿到高质量反馈
- 更低成本沉淀内容资产
- 找到愿意共创或传播的人
- 把信任接成转化或长期口碑

## Use This Skill When

- 要做社区
- 要做 ambassador
- 要做渠道商、服务商、分销商体系
- 要做 Discord / Slack / Forum
- 要做用户共创和口碑传播

## Core Rules

### 1. 先找对人

社区负责人最好同时满足：

- 是产品的重度用户
- 能组织活动
- 能产出内容

### 2. 给对价值

用户 / 伙伴进来后，至少给其中一种：

- 名
- 利
- 资源
- 身份
- 更快的反馈

### 3. 让社区可运转

要准备：

- 招募文案
- 活动节奏
- 内容包
- 权益机制
- 分成或激励机制

如果社区还处在早期，优先做“少数高质量用户的深关系”，不是先堆人数。

## Output Format

输出时给：

1. 社区对象是谁
2. 社区在当前阶段的作用
3. 招募标准
4. 激励设计
5. 运营节奏
6. 内容物料
7. 关键指标

## Guardrails

- 不要先堆人数
- 不要没有内容就拉人
- 不要让社区只剩客服功能
- 不要忽视长期维护
- 不要把所有用户都当成 ambassador 候选人
